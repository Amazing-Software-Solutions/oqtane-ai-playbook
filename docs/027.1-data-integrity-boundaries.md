# 027.1 - Data Integrity Boundaries

Oqtane modules rely on **consistent, invariant data**.

All persisted data must represent a **single source of truth**, independent of user, locale, or execution context.

Violating this principle introduces silent data corruption that is difficult to detect and impossible to reason about reliably.

---

## The Core Rule

> **Persist canonical data.
> Never persist contextual representations.**

If a value would differ depending on who retrieves it or how it is displayed, it must not be stored in that form.

---

## Canonical vs Contextual Data

### Canonical Data (Allowed)

Canonical data is:

- invariant across users and environments
- independent of presentation concerns
- safe to persist as the source of truth

Examples include:

- UTC-based timestamps
- numeric values without formatting
- base units of measurement

---

### Contextual Data (Prohibited for Persistence)

Contextual data depends on:

- user preferences
- timezone
- culture or formatting
- request or environment context

Contextual representations are valid for display, but never for storage.

---

## Persistence Requirements

All persisted values MUST:

- have a consistent meaning for all users
- be independent of presentation logic
- remain stable regardless of request context
- represent the same truth across all environments

---

## Boundary Violations (Reject on Sight)

The following patterns must be rejected immediately:

- Persisting localized or formatted values
- Applying timezone conversions before storage
- Using user preferences to shape stored data
- Storing values derived from request-specific context
- Allowing site or tenant presentation settings to influence persistence

---

## Examples

### Invalid - Contextual Persistence (Time)

```csharp
entity.CreatedOn=utcDate.ToLocalTime();
```

---

### Invalid - Contextual Persistence (Culture)

```csharp
entity.Price=price.ToString("C");
```

---

### Invalid - Contextual Persistence (User Preference)

```csharp
entity.Distance=ConvertToMiles(distanceKm, userPreference);
```

---

### Valid - Canonical Persistence

```csharp
entity.CreatedOn=utcDate;
entity.Price=price;
entity.DistanceKm=distanceKm;
```

---

## Relationship to Other Rules

- Complements **017 Client / Server Responsibility Boundaries**
- Defines what is allowed to be persisted within those boundaries

This rule governs **data integrity**, not execution responsibility.

---

## Summary

- Persist invariant data only
- Never store context-dependent values
- Separate storage from presentation

Data integrity is foundational.

Once compromised, all downstream behaviour becomes unreliable.
